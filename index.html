Get on FAB

Getting Started

Installation

Gridline Blueprints installs from FAB into your Unreal Engine project with zero configuration required. Every feature is toggleable in Editor Preferences.

1 · Install from FAB

Open the FAB launcher link or find Gridline Blueprints on the FAB marketplace. Install it to your engine version (5.6 or 5.7).

2 · Enable the Plugin

In Unreal Editor, go to Edit → Plugins, search for Gridline Blueprints, enable it, and restart the editor. Orthogonal routing activates immediately.

3 · Configure

Open Editor Preferences and find sections under Plugins: Gridline Wire Routing, Gridline Nodes, and Gridline Context Menu. Every setting has a tooltip and sensible defaults.

Zero runtime cost. All four modules load as Editor or UncookedOnly types. Nothing ships with your packaged build.

Module · GridlineWireRouting

Orthogonal Wire Routing

Replaces Unreal's default spline wires with clean, right-angle connections. The routing pipeline processes every wire through a series of phases — U-turn resolution, wire spacing, node avoidance, corner processing, and junction detection — producing organized, readable graphs.

How It Works

A custom FConnectionDrawingPolicy factory is registered at module startup via FEdGraphUtilities::RegisterVisualPinConnectionFactory. When the graph editor draws connections, it creates a FGridlineBlueprintConnectionDrawingPolicy instead of the default Kismet policy.

Each wire is stored as a FWireData struct containing start/end positions, associated pins, and an array of FWireSegment segments. The pipeline runs five phases in order:

  • UTurnResolutionPhase — Detects wires where the destination is behind the source and converts them into 5-segment U-turn paths routed around node bounds.
  • WireSpacingPhase — Groups overlapping parallel segments using disjoint-set union, sorts them by flow direction, and offsets them to produce even spacing.
  • NodeAvoidancePhase — Detects wires that pass through unrelated nodes and reroutes them with detour segments (Complex) or sideways shifts (Simple).
  • CornerProcessingPhase — Replaces sharp 90° turns with Bézier arcs (Eased/Fillet) or diagonal cuts (Chamfer), and handles tight-turn segments with splines or straight lines.
  • JunctionPhase — Finds perpendicular wire crossings using spatial bucketing and inserts visual junction indicators (Jump arcs, Ring circles, Disc fills, or Break gaps).

Wire rendering uses FSlateDrawElement::MakeDrawSpaceSpline for all segment types, including hover detection for line and curved segments to support wire selection.

Wire Spacing

Overlapping parallel segments are detected using a disjoint-set (union-find) algorithm that groups segments sharing the same axis position and overlapping ranges. Each group is sorted by flow direction and source/destination position, then offset by even increments. The phase runs multiple passes to resolve cascading overlaps.

SettingDefaultRangeDescription
Enable Wire Spacingtrue—Offset overlapping parallel segments so individual wires remain visible.
Wire Spacing168 – 64Pixel distance between adjacent parallel wires in a group.
Space Shared Source Wiresfalse—When enabled, wires from the same output pin are spaced apart. When disabled, they share the same path until they diverge.
Space Shared Destination Wiresfalse—When enabled, wires going to the same input pin are spaced apart. When disabled, they share the same path segment.

Node Avoidance

The avoidance phase collects all node bounds (excluding source, destination, and comment nodes) and checks whether the routable segments of each wire intersect any obstacle. Two strategies are available:

  • Simple — Shifts the vertical segment sideways to clear the union of all overlapping obstacles. Fast, but can produce longer wires.
  • Complex — Iteratively inserts additional horizontal/vertical segments to route fully around each obstacle, processing them in order of encounter. Produces tighter paths at the cost of more segments.

Horizontal single-segment wires (same Y) also receive avoidance: if an obstacle is detected, the wire is temporarily split into a 3-segment path for rerouting.

Junctions

Junctions mark where wires visually cross. The phase uses spatial bucketing (64 px buckets) to efficiently find intersections between horizontal and vertical segments from different wires. Wires sharing the same source or destination pin are excluded. Each crossing inserts a Junction segment that the renderer draws in the chosen style.

Flow Animation

Animated indicators travel along wires to visualize data flow direction. The animation system traverses the segment array using cumulative distance, sampling position and direction at each point. Two styles are available:

  • Dot — Draws bubble images at regular intervals along the wire using FSlateDrawElement::MakeBox. Dots skip junction segments to avoid visual clutter.
  • Pulse — Renders a multi-layer glow wave using custom vertex meshes (MakeCustomVerts). Each pulse is 60% of the animation spacing. Brightness fades sinusoidally from the center of each pulse.

Module · GridlineNamedReroutes

Named Reroute Nodes

A Declaration/Usage node system that lets you capture a value once and reference it anywhere in the Blueprint — like named variables on the graph canvas. Eliminates long-distance spaghetti wires and makes complex graphs dramatically easier to read.

Declaration Node

Captures an input value and makes it available by name. Has an input pin (receives the wire) and an output pin (structural link to usage nodes). Supports renaming, custom color, copy/paste, undo/redo, and duplicate detection.

Usage Node

References a Declaration by GUID. Place as many as needed anywhere in the Blueprint. The output pin mirrors the Declaration's type and propagates changes automatically. Jump-to-definition navigates to the Declaration.

Internal Architecture

  • GUID Binding — Each Declaration gets a unique FGuid (VariableGuid). Usage nodes store this GUID in DeclarationGuid and resolve the reference at load, paste, and undo via FindDeclarationByGuidInBlueprint.
  • Pin Type Propagation — When the Declaration's input is connected, its pin type propagates to the output pin and all Usage nodes via PropagatePinType. Disconnecting all wires resets to Wildcard.
  • Structural Pin — Each Usage node has a hidden input pin (StructuralInputPin) linked to the Declaration's output. This creates a dependency edge for the compiler without being visible in the graph.
  • Compiler Expansion — During ExpandNode, the Declaration traces through its input chain (handling nested Knots and other Declarations via TraceToRealInput) and reconnects all consumers directly to the real source pin. Execution pins get an intermediate Knot passthrough.
  • Name Uniqueness — MakeNameUnique scans all graphs in the Blueprint and appends numeric suffixes if a name collision is detected.
  • Action Database — Usage nodes register themselves in GetMenuActions with a DynamicUiSignatureGetter that displays the current Declaration name in the context menu under "Named Reroutes". The action is filtered by Blueprint scope via IsActionFilteredOut.

Context Menu Actions

Right-click context menus are extended via UToolMenus for three node types:

  • K2Node_Knot → Convert to Named Reroute — Replaces the reroute with a Declaration (and optionally a Usage for outgoing connections), preserving all wiring and resolved pin types.
  • NamedRerouteDeclaration → Select Usages, Create Usage, Convert to Reroute — Find all references via the search panel, spawn a new Usage node, or revert to a standard Knot.
  • NamedRerouteUsage → Select Declaration, Convert to Reroute — Navigate to the source Declaration or revert the entire Declaration+Usage chain to Knots.

Plugin Disable Safety

The module watches the .uproject file for changes using IDirectoryWatcher. If Gridline Blueprints is set to "Enabled": false, a dialog prompts the user to convert all Named Reroute nodes in the project to standard Reroute nodes before the plugin becomes inactive. This prevents broken Blueprint references. Conversion scans all Blueprints under /Game via the Asset Registry and calls ConvertToRerouteNode on every Declaration.

Custom Visuals

A custom FGraphPanelNodeFactory provides specialized Slate widgets:

  • SGraphNodeNamedRerouteDeclaration — Uses a custom output pin widget (SGraphPinDeclarationOutput) that only shows as "connected" when genuine (non-structural) connections exist, avoiding misleading visuals from hidden structural links.
  • SGraphNodeNamedRerouteUsage — Hides the structural input pin from the widget, showing only the output pin for a clean appearance.

Module · GridlineContextMenu

Context Menu Enhancements

Four independently toggleable features that improve the Blueprint right-click context menu. Each feature implements the IGridlineFeature interface and is managed by the module via a descriptor table. Features can be enabled/disabled at runtime through Editor Preferences with immediate effect.

Fuzzy Search

Injects additional keywords into every UBlueprintNodeSpawner to enable fuzzy matching. Generates spaceless concatenations, acronym initialisms (e.g., gpc → GetPlayerController), and transposition/deletion misspelling variants. Keywords are processed in batches of 200 per tick to avoid editor hitches.

Auto-Expand Categories

Automatically expands all top-level categories when the context menu opens. Uses a reinterpret_cast accessor to set bAutoExpandActionMenu = true on the SGraphActionMenu, then triggers a full tree rebuild.

Menu Resize

Patches the ancestor SBox of the SGraphActionMenu with custom width and height overrides. Searches up to 8 ancestor levels. Uses a tag (GridlineMenuResize_WxH) to avoid redundant patches and restores original sizes on shutdown.

Recent Nodes

Tracks recently placed Blueprint nodes by subscribing to UEdGraph::OnGraphChanged for every graph in opened Blueprints. Injects a "Recently Used" section above the filter results. Press the Cycle Key while the menu is open to fill the search box with recent entries. Supports forward/reverse cycling with Shift or a dedicated reverse key.

Feature Lifecycle

All context menu features follow two base classes:

  • FKeywordInjectorBase — For features that modify spawner keywords (Fuzzy Search). Subscribes to FBlueprintActionDatabase::OnEntryUpdated, batch-processes all existing spawners at startup (200/tick), stores original keywords in a shared static map, and restores them on shutdown. Supports ReApply for hot-reload scenarios.
  • FSlateMenuFeature — For features that interact with the context menu widget (Auto-Expand, Menu Resize, Recent Nodes). Listens to FSlateApplication::OnFocusChanging to detect when a menu-type window opens containing an SGraphActionMenu. Caches the reference and provides per-tick callbacks. Automatically cleans up on window close.

Module · GridlineNodes

Node Alignment

Automatically arranges selected nodes and their connected execution graphs into a clean, grid-snapped layout with a single keypress. The alignment system processes execution chains, data input chains, and independent subgraphs to produce organized, readable layouts.

How It Works

A global IInputProcessor intercepts key events when the active widget is an SGraphPanel with selected nodes. The alignment pipeline uses a phase-based architecture:

  • Seed Collection — Identifies execution-pin-bearing nodes in the selection. Pure data nodes trace forward to find their nearest execution node.
  • Flood Fill — Groups connected execution nodes into independent subgraphs, each processed separately.
  • Anchor Node — The topmost-leftmost execution node with no selected predecessors is chosen as the anchor. Ties are broken by downstream pin index. The anchor is grid-snapped and claimed.
  • Recursive Layout — Starting from the anchor, the algorithm recursively lays out:
    • Data inputs — Pure data nodes are placed to the left of their consuming execution node, chaining recursively.
    • Execution outputs — Successor nodes are placed to the right, with branches stacked vertically. The first branch can be wire-straightened (pin-to-pin Y alignment).
    • Execution inputs — Predecessor nodes below the current node.
  • Overlap Resolution — ResolveChainOverlaps shifts subtrees downward by grid increments until no grid-cell overlaps remain. ShiftGroupUntilClearX pushes groups rightward to avoid predecessor overlap.
  • Group Positioning — Controlled by Group Alignment Mode: Right of Predecessor places the group strictly right, while Left with Predecessor aligns left edges first, then shifts to clear.

The grid size is 8× the Blueprint snap grid (e.g., 128 px at default zoom). All positions are snapped to this ruler grid.

Technical Overview

Architecture

Gridline Blueprints is composed of four independent modules, each handling a distinct concern. All modules are editor-only with zero runtime footprint.

GridlineWireRouting

Type: Editor
Core orthogonal wire rendering. Registers a custom FGraphPanelPinConnectionFactory that intercepts all Blueprint and Animation graph connection drawing. Contains the 5-phase wire processing pipeline, segment rendering, hover detection, hidden reroute node logic, reroute node flipping, wire slicing support (UE 5.7), and flow animation.

GridlineNamedReroutes

Type: UncookedOnly
Named Reroute Declaration and Usage K2 nodes. Provides custom UCLASS nodes, compiler expansion, context menu extensions via UToolMenus, custom Slate node/pin widgets via FGraphPanelNodeFactory, and the .uproject file watcher for safe plugin disable conversion.

GridlineContextMenu

Type: Editor
Context menu feature framework. Four features (Fuzzy Search, Auto-Expand, Menu Resize, Recent Nodes) implemented against a common interface. Features are registered from a descriptor table and support hot-toggle via settings change delegates.

GridlineNodes

Type: Editor
Node alignment system. Global input preprocessor, phase-based layout pipeline, grid-cell claiming for overlap resolution, and the Node Customization phase placeholder for future visual enhancements.

Key Design Principles

  • Phase-Based Pipelines — Both wire routing and node alignment use ordered phase arrays (IWireRoutingPhase, INodeLayoutPhase). Each phase reads and mutates a shared context, making it easy to add, remove, or reorder processing steps.
  • Non-Destructive — Keyword injectors store original values and restore them on shutdown. Named Reroute conversions use scoped transactions for full undo support. SBox patches track and revert overrides.
  • Lazy Batch Processing — Keyword injection processes spawners in batches of 200 per engine tick to avoid blocking the editor during startup with large action databases.
  • Weak References — Spawner maps, widget caches, and graph handles all use TWeakObjectPtr or TWeakPtr to prevent dangling pointers when assets are unloaded or widgets destroyed.
  • Settings are Live — All three settings classes (UGridlineWireRoutingSettings, UGridlineNodesSettings, UGridlineContextMenuSettings) derive from UDeveloperSettings with config = EditorPerProjectUserSettings. Changes in Editor Preferences take effect immediately without restart.

Platform Support

Compatibility

Gridline Blueprints is tested and supported across the following configurations.

Unreal Engine

Supports UE 5.6 and UE 5.7. Wire slicing (new in 5.7) is supported via compile-time version guards (ENGINE_MAJOR_VERSION / ENGINE_MINOR_VERSION).

Platforms

Windows (Win64), macOS (Mac), and Linux. All four modules specify PlatformAllowList for these three targets.

Graph Types

Blueprint Graphs (Event Graphs, Function Graphs) and Animation Blueprint Graphs. Each can be independently enabled or disabled in settings.

Editor-Only

Three modules use Type: Editor, one uses Type: UncookedOnly. Nothing is included in packaged builds. Zero runtime overhead.

Blueprint Assist

Compatible with the Blueprint Assist plugin. Both can run simultaneously — Blueprint Assist handles its own formatting while Gridline Blueprints handles wire routing and its own alignment features independently.

Safe to Disable

Disabling the plugin in your .uproject triggers an automatic prompt to convert Named Reroute nodes to standard Reroute nodes, preventing broken references. All other features are purely visual and leave no traces when disabled.

Support

Frequently Asked Questions

Common questions about setup, behavior, and troubleshooting.

Does Gridline Blueprints affect my packaged game?
No. All four modules are Editor or UncookedOnly type. They are excluded from cooked builds entirely. There is zero runtime overhead — no code, no assets, and no dependencies ship with your game.
Can I use this with Animation Blueprints?
Yes. Wire Routing and Node Alignment both have independent "Enable for Animation Graphs" toggles. The drawing policy detects UAnimationGraphSchema and applies the same orthogonal routing pipeline.
What happens to Named Reroute nodes if I disable the plugin?
The plugin watches your .uproject file for changes. When it detects the plugin being disabled, it prompts you to convert all Named Reroute Declaration/Usage nodes to standard Reroute (Knot) nodes. If you choose "Yes," all Blueprints under /Game are scanned, converted, and saved. If you choose "No," the nodes remain but won't function until the plugin is re-enabled.
How does Fuzzy Search work? Does it replace the search algorithm?
It does not replace the search algorithm. Instead, it pre-processes every UBlueprintNodeSpawner in the action database and appends additional keywords to the existing keyword field. This means the engine's built-in search matches against a richer set of terms: spaceless concatenations (getplayercontroller), acronym initialisms (gpc), and common misspelling variants (character transpositions and deletions). Original keywords are preserved and restored when the feature is disabled.
Will the alignment system rearrange nodes I didn't select?
When using the full alignment keybind (F by default), the system expands from your selection to include the full connected execution graphs. Use the selected-only keybind (G by default) to align only the nodes you've explicitly selected without expanding.
Can I use this alongside Blueprint Assist?
Yes. Gridline Blueprints and Blueprint Assist are compatible and can run simultaneously. Blueprint Assist handles its own node formatting, while Gridline Blueprints provides orthogonal wire routing, Named Reroute nodes, context menu features, and its own alignment system independently.
Why do some wires still use splines instead of orthogonal routing?
Short wires (below the Short Wire Threshold, default 120 px) use a simplified style (Ortho, Spline, or Straight) instead of the full routing pipeline. This is intentional — very close pins look cleaner without full right-angle routing. You can adjust or disable this by setting the threshold to 0. Additionally, tight turns (vertical segments too short for corners) may display as splines or straight lines depending on your Tight Turn settings.
How do I reset all settings to defaults?
Settings are stored in EditorPerProjectUserSettings. Delete the relevant section from your project's Saved/Config/ folder (the EditorPerProjectUserSettings.ini file) or manually reset each value in Editor Preferences. The plugin uses sensible defaults for all settings.
The Menu Resize feature isn't working. What should I check?
Menu Resize works by finding the SBox ancestor of the SGraphActionMenu widget, searching up to 8 levels. If your theme or another plugin modifies the context menu hierarchy, the SBox may be further up. Check the Output Log for a warning mentioning "no SBox ancestor located." If this occurs, the MaxAncestorSearchDepth constant in the source may need adjustment. Please report the issue so it can be addressed in a future update.
Can Named Reroutes carry execution (white) wires?
Yes. Named Reroute nodes support both data and execution pin types. The pin type is resolved from whatever is connected — if the Declaration's input receives an execution wire, the type propagates to all Usage nodes. During compilation, execution-type Declarations insert an intermediate Knot passthrough node to maintain the execution flow.
