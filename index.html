Introduction
What is Gridline Blueprints?

A purely editor-side Unreal Engine plugin that replaces the default curved splines in Blueprint and Animation graphs with precise orthogonal (right-angle) wires ‚Äî and adds a full suite of visual quality-of-life features on top.

Getting Started
Installation

Gridline Blueprints installs like any standard UE Marketplace or Fab plugin. No source build is required.

1

Acquire the Plugin

Purchase and download Gridline Blueprints from Fab via the Epic Games Launcher. The plugin will appear in your Fab Library.

2

Install to Engine or Project

In the Epic Games Launcher, click Install to Engine next to Gridline Blueprints and select your target engine version (5.6 or 5.7). Alternatively, copy the GridlineBlueprints folder into your project's Plugins/ directory for a project-local install.

3

Enable the Plugin

Open your project in the Unreal Editor, then go to Edit ‚Üí Plugins. Search for Gridline Blueprints under the Blueprint category and check the Enable checkbox.

Note: The editor will prompt you to restart. Save your work and click Restart Now.
4

Verify the Plugin is Active

After the restart, open any Blueprint. Wires should now route orthogonally. If they still appear as curved splines, proceed to the troubleshooting section or the FAQ below.

5

Configure Settings

All plugin settings are found at Edit ‚Üí Project Settings ‚Üí Plugins ‚Üí Gridline Blueprints. Settings are stored per-user in EditorPerProjectUserSettings and are not committed to source control by default.

First Steps
Quick Start Guide

Get up and running with the most impactful settings in under two minutes.

1

Open a Blueprint

Open any Blueprint asset. Your wires will immediately route orthogonally using the default settings ‚Äî Destination bias, 28px pin length, eased corners, wire spacing enabled, and node avoidance on.

2

Choose Your Routing Bias

In Project Settings, under Routing ‚Üí Wire Routing Bias, choose:

  • Destination ‚Äî the vertical bend appears near the input pin. Good for left-to-right flow graphs.
  • Source ‚Äî the vertical bend appears near the output pin.
3

Adjust Corner Style

Under Appearance ‚Üí Corner Style, pick the look you prefer:

  • Eased ‚Äî smooth B√©zier arc (default, most polished)
  • Fillet ‚Äî precise quarter-circle arc
  • Chamfer ‚Äî sharp diagonal cut, industrial look

Increase Corner Size (0‚Äì32 px) to control how large the curve is.

4

Try Wire Spacing

Wire Spacing is enabled by default at 16 px. If you have many parallel wires, try increasing this to 20‚Äì24 px so each wire is clearly distinguishable. Decrease it for denser graphs.

5

Place Your First Named Reroute

Right-click any Reroute node in your Blueprint and select Convert to Named Reroute. Give the declaration a descriptive name. Then right-click anywhere in the graph and search for that name under Named Reroutes to place a Usage node ‚Äî eliminating long wires that span large distances.

Wire Spacing

When multiple wires share the same axis and overlap spatially, Wire Spacing offsets them perpendicular to their direction so each wire remains individually visible and selectable.

SettingDefaultDescription

Wire Spacing is applied in two passes ‚Äî once after initial routing and once after node avoidance ‚Äî so freshly rerouted wires are also spaced correctly. The sort order of wires within a group is determined by their source position to keep the visual arrangement consistent.

Node Avoidance

Node Avoidance detects when a wire's routable segments pass through an unrelated node's bounding rectangle and reroutes the wire around it.

SettingDefaultDescription
Enable Node AvoidancetrueEnable or disable the node avoidance pass entirely.
Node Avoidance StyleComplexSimple shifts the vertical segment sideways to clear the obstacle. Complex inserts additional horizontal and vertical detour segments to route entirely around the node bounds.
Node Avoidance Padding16 pxMinimum clearance between rerouted wire paths and node edges. Range: 0‚Äì64 px.
Note: Comment nodes are explicitly excluded from avoidance checks ‚Äî wires are allowed to cross comment box borders freely.

The avoidance phase runs up to five passes per wire to handle multiple stacked obstacles. Source and destination nodes are always excluded from their own wire's avoidance check.

Feature Deep-Dive
Named Reroutes

Named Reroutes let you label a wire value once ‚Äî the Declaration ‚Äî and consume it from any number of Usage nodes anywhere in the same Blueprint, eliminating long wires that span large graph areas.

Declaration Node

A Named Reroute Declaration node has one input pin and one output pin, just like a standard Reroute node. It is distinguished by a colored title bar bearing the reroute's name, set as a property you can edit directly in the node.

  • Name ‚Äî displayed on the node title bar and used to identify the reroute in Usage node spawners. Names are enforced to be unique within a Blueprint; a numeric suffix is appended automatically if needed.
  • Node Color ‚Äî the color of the title bar. Both the Declaration and all its Usage nodes share this color for visual grouping.
  • The Declaration's input pin connects type information upstream. If the input is unconnected but a Usage's output is typed, the type propagates back to the Declaration automatically.
  • Double-clicking a Declaration (Jump to Definition) opens the Find Results panel filtered to all Usages of that name within the Blueprint.

Usage Node

A Named Reroute Usage node has only an output pin (visible) plus a hidden structural input pin that links it to the Declaration's output for type inference. It behaves like a pure node that re-emits the Declaration's value.

  • Place via right-click ‚Üí search the declaration's name under the Named Reroutes category.
  • Double-clicking a Usage (Jump to Definition) navigates to its Declaration node, selecting and zooming to it.
  • Multiple Usage nodes per Declaration are fully supported.
  • If the Declaration is deleted, Usage nodes become orphaned ‚Äî they display a black title and the label Invalid Named Reroute. See the FAQ for recovery options.

Compilation Behaviour

At compile time, Named Reroute nodes are transparent to the Blueprint compiler ‚Äî the compiler sees the original source pin, not the reroute pair. This is achieved through ExpandNode:

  • The Declaration traces its input chain upstream through Knots and other Declarations to find the first real source pin.
  • All consumers (direct or via Usage nodes) are reconnected directly to that source pin in the intermediate graph.
  • The structural input pin on each Usage node (the hidden link to the Declaration's output) is broken before compilation so it never appears in the compiled result.
  • If a Declaration is chained from another Declaration and the chain contains a cycle, compilation will emit a warning and leave those consumers disconnected.
Technical
Architecture

An overview of how the plugin integrates with the Unreal Editor graph system and how the wire routing pipeline is structured.

Module Structure

FGridlineBlueprintsModule

Top-level IModuleInterface. Registers the connection factory and node factory on startup; unregisters on shutdown.

FGridlineBlueprintsConnectionDrawingPolicyFactory

FGraphPanelPinConnectionFactory subclass. Intercepts schema queries for K2 and AnimGraph schemas, returning the custom drawing policy.

FGridlineBlueprintsConnectionDrawingPolicy

FKismetConnectionDrawingPolicy subclass. Collects wire data, runs the routing pipeline, and draws segments.

UGridlineBlueprintsSettings

UDeveloperSettings subclass. Config class stored in EditorPerProjectUserSettings. Drives all phase behaviour.

IWireRoutingPhase

Abstract interface. Each routing stage is an independent IWireRoutingPhase implementation operating on the shared TArray<FWireData>.

Named Reroute Nodes

UK2Node_NamedRerouteDeclaration and UK2Node_NamedRerouteUsage ‚Äî UK2Node subclasses. Registered via FNamedRerouteNodeFactory (FGraphPanelNodeFactory).

Wire Routing Pipeline

Basic Path(inline)Generates the initial 3-segment orthogonal path (H‚ÄìV‚ÄìH) for each wire based on routing bias and pin length.

Wire Data Model

Each connection is represented by an FWireData struct that flows through the pipeline:

struct FWireData {
  int32 LayerId;
  FVector2f Start, End;
  FConnectionParams Params;
  TArray<FWireSegment> Segments; // mutated by each phase
  bool bIsUTurn, bWasNodeAvoided, bIsExecWire, bIsForcedNodeAvoided;
};

enum class EWireSegmentType : uint8 {
  Line, Corner, Junction, Spline
};

Phases may add, remove, or modify FWireSegment entries. The final segment array is consumed by the draw helpers (DrawLine, DrawCorner, DrawJunction, DrawSpline) and hover detection.

UE 5.7 Slice Line Support

The plugin includes conditional compilation blocks for the slice-line API introduced in UE 5.7 (ENGINE_MINOR_VERSION >= 7). When active, any wire whose segments intersect the slice line is registered in ConnectionsIntersectingSliceLine, which triggers Unreal's built-in de-emphasis alpha. This is handled for both full orthogonal wires and short-wire paths.

Blueprint Assist compatibility: Blueprint Assist operates at the node layout layer (repositioning nodes) while Gridline Blueprints operates at the wire drawing layer (rendering connections). The two systems do not conflict at runtime. However, because Blueprint Assist may reposition nodes after Gridline's wire spacing has been calculated, visual spacing artifacts can occasionally appear until the graph is re-rendered. Both plugins can be used together productively.
Material / Niagara / PCG graphs: These editors use different graph schemas (not UEdGraphSchema_K2 or UAnimationGraphSchema). The plugin's connection factory explicitly checks for K2 and Animation schemas and returns nullptr for all others, so those editors are completely unaffected.
